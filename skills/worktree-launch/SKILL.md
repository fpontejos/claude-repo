# Worktree Launch: $ARGUMENTS

Create an isolated git worktree at `.worktrees/<name>/` in the current project, pre-configured for development. Project-specific behavior (ports, env files, dependency fixups, smoke tests) is driven by an optional config file at `.claude/worktree.md`; without it, the skill runs a safe generic path.

**Scope note:** This is the generic, project-agnostic protocol. If the current project has its own worktree skill (e.g. `/launch-worktree` in the CableGenie repos, `/worktree-setup` in SimuCable), use that instead — it encodes repo-specific fixups this skill knows nothing about.

**Branching authorization:** This skill creates a branch by design. The user invoking it IS the explicit authorization required by `~/.claude/rules/git-workflow.md` — do not re-ask.

## Usage

```
/worktree-launch <branch-name>
```

Argument: branch name (e.g. `feat/report-redesign`). Worktree dirname = last `/`-segment of the branch name.

Argument provided: "$ARGUMENTS"

---

## Step 0: Read Project Config

Check for `.claude/worktree.md` at the repo root (see the Config Contract section below).

- **Present:** Read it. Its sections (`## Ports`, `## Env template`, `## Post-create`, `## Smoke test`) enable the optional steps below.
- **Absent:** Run the generic-only path (Steps 1, 2, 5, 6) and, in the final report, offer to scaffold `.claude/worktree.md` from the template below.

## Step 1: Parse & Validate

Parse `$ARGUMENTS` to extract the branch name. `<name>` = last `/`-segment.

**Guard clauses:**

```bash
git worktree list --porcelain | grep -c '<name>' || true
git branch --list '<branch>' | head -1
```

- Worktree already exists at `.worktrees/<name>/` → **stop** and report: "Worktree already exists. Use it directly or remove it first with `/worktree-teardown <name>`."
- Branch exists but no worktree → ask the user: create the worktree from the existing branch, or create fresh from the default branch?
- Neither exists → proceed.

## Step 2: Create the Worktree

Verify `.worktrees/` is gitignored; if not, add it to `.gitignore` and mention this in the report.

```bash
git worktree add .worktrees/<name> -b <branch>
```

## Step 3: Assign Ports (config: `## Ports`)

Skip if the config has no `## Ports` section.

Each worktree gets a unique offset from the base ports declared in config.

1. List existing assignments: `grep -h '_PORT=' .worktrees/*/.env.worktree 2>/dev/null`
2. Find the lowest unused offset in increments of 10 (+10, +20, +30, …)
3. Skip any ports listed as reserved in the config

## Step 4: Create `.env.worktree` (config: `## Env template`)

Skip if the config has no `## Env template` section.

Write the template to `.worktrees/<name>/.env.worktree`, substituting placeholder variables (e.g. `$BACKEND_PORT`, `$FRONTEND_PORT`) with the ports assigned in Step 3. Verify by reading the file back.

## Step 5: Project Post-Create Steps (config: `## Post-create`)

Skip if the config has no `## Post-create` section.

Run the listed steps in order. Typical contents: dependency path fixups for the extra directory depth, lockfile deletion + re-sync, package installs. Treat these steps as authoritative — they encode the project's known traps.

## Step 6: Patch CLAUDE.md with Worktree Header

If the worktree contains a `CLAUDE.md`, prepend this block at the very top (before the existing heading), including only the rows that apply. If the project has no `CLAUDE.md`, skip and note it in the report.

```markdown
<!-- WORKTREE CONTEXT — auto-generated by /worktree-launch -->
> **This is a worktree checkout**, not the main repo.
>
> | Property | Value |
> |----------|-------|
> | Branch | `<branch>` |
> | Worktree path | `.worktrees/<name>/` |
> | Ports | <assigned ports, if any> |
>
> **Constraints:**
> - Never `cd` into subdirectories in orchestrator bash calls — use `--project`/`-C`-style flags or subshells
> - <project-specific constraints from `.claude/worktree.md`, if it declares any>
<!-- END WORKTREE CONTEXT -->
```

The HTML-comment markers let `/worktree-merge` find and strip the block before merging — the header must never reach the default branch.

## Step 7: Smoke Test (config: `## Smoke test`)

Skip if the config has no `## Smoke test` section.

Run the configured command and compare against the expected output. If it fails, report the failure — do not declare the worktree ready.

## Step 8: Report

```
Worktree ready: .worktrees/<name>/
Branch:         <branch>
Ports:          <assigned, or "n/a (no port config)">
Config:         .claude/worktree.md <found | not found — generic path used>
Post-create:    <steps run, or "none">
Smoke test:     <result, or "not configured">

Next steps:
- /worktree-merge <name> when the feature is done
- /worktree-teardown <name> to discard
```

If no config was found, offer: "This project has no `.claude/worktree.md`. Scaffold one from the template so future worktrees get ports/env/setup automatically?"

## Config Contract: `.claude/worktree.md`

Per-project file, all sections optional. Scaffold template:

```markdown
# Worktree Config

Read by the user-level /worktree-launch, /worktree-merge, /worktree-teardown skills.

## Ports

- Backend base: 8000
- Frontend base: 5173
- Reserved: 8000, 5173 (main dev), 8001, 5174 (E2E)

## Env template

    # Dev server ports (worktree-isolated)
    BACKEND_PORT=$BACKEND_PORT
    FRONTEND_PORT=$FRONTEND_PORT

## Post-create

1. <dependency path fixup command, if editable deps use relative paths>
2. <delete cached lockfile + re-sync, if paths were rewritten>
3. <install commands, e.g. npm install in a subshell>

## Test command

    <command /worktree-merge runs pre- and post-merge, e.g. uv run pytest tests/>

## Smoke test

    <command + expected output, e.g. curl health endpoint returns 200>

## Teardown extras

    <cleanup /worktree-teardown runs first, e.g. docker compose down -v>
```

## Known Gotchas (generic)

- **`cd` persists across Bash calls in this harness.** Never `cd` into the worktree in an orchestrator bash call — use `git -C`, `uv run --project`, or subshell parentheses `(cd … && …)`.
- **Worktrees are created from branch HEAD.** Uncommitted files in the main checkout (in-progress plans, drafts) are absent. Copy them in manually if the feature work needs them.
- **Lockfiles cache resolved paths.** If Post-create rewrites dependency paths, the project's lockfile must be deleted before re-syncing, or the old paths are silently reused.
- **Don't reuse port offsets.** Always scan existing `.worktrees/*/.env.worktree` before assigning.
- **Trust `git worktree list`, not `ls`.** Stray directories under `.worktrees/` may not be registered worktrees.
